SEALIGHT · 首页 v04 · 方向说明

首页改版的三个决策，分别讲明白。

v04 不是一个首页方案 —— 是三个独立的决策轴。导航架构决定用户怎么进漏斗，Hero 决定品牌语气，色调决定视觉情绪。这份文档把每个轴的选项拆开讲：它是什么、为什么这样、适合谁、风险在哪。读完你可以自己组合出首选方案， 也可以直接采纳我们推荐的那组。

01 · 把首页拆成三根独立的轴

v03 的问题不是"不好看"，是几个决策被绑在一起——想换 Hero 就被迫连品牌语气一起换，想调导航就牵动产品区配色。v04 把决策解耦： 每一轴都能单独投票，最后再合。对评审团队来说，会议从"你喜欢哪个方案"变成"这三件事你各站哪一边"。

导航架构
决定首页 → 产品页的进入方式。按需求 / 按车型 / 极简三种逻辑。
Hero 方向
决定首屏品牌语气。电影感（工程叙事）/ 杂志风（生活方式）/ 适配优先（工具承诺）。
色调系统
决定整页视觉情绪。Midnight 深色影院感 / Daylight 浅色编辑风 / Hybrid 混排。

02 · 导航架构：三种让用户进漏斗的方式

北美 DIY 车主到店里最想问的第一句是"这玩意我的车能不能用"。三版导航的差异本质上是在哪一步把这个问题问出来——在顶部常驻 pill、在专属适配栏、还是在右侧抽屉里。

Nav A · 按需求分类的 Mega 菜单

NAV · A

顶栏三个主入口：按需求、按车型、按灯泡型号。鼠标悬停任一项展开三列 mega，第四列固定是当季旗舰（Nova）推荐。 右上角是一枚常驻的 2022 · Ford · F-150 适配 pill，用户随时能改。

适合谁
想先逛再决定的用户——不确定自己要"亮度"还是"防眩"，也不想立刻选车型。
核心承诺
三种购物路径一视同仁：按痛点选、按车型选、按零件号选。
信息密度
中到高。Mega 展开时覆盖约 380px 垂直空间。

优势

  • 探索友好，三条漏斗平权
  • Nova 旗舰在第四列长期曝光
  • 保留常驻 fitment pill，不损失效率

权衡

  • 移动端 mega 需重新设计（底抽屉）
  • 对老用户偏冗余——他们只想点 F-150
  • 三入口需要精确命名，否则用户不知道点哪个
Nav A · Mega 菜单（鼠标悬停查看下拉）

Nav B · 适配优先的双层栏

NAV · B ★

顶层是产品大类（Headlights / Fog / Interior / Off-road / Combo / Support），下面是一条专属的Year · Make · Model 选择栏，右边实时显示"12 matches"。 把 SEALIGHT 最核心的承诺（精准适配）直接做成 chrome。

适合谁
已经知道要换什么、只差确认车型的老用户——也就是 Amazon 习惯过来的那拨。
核心承诺
"你的车装哪个我知道"。把适配做成视觉签名。
信息密度
高。顶栏 60px + 适配栏 52px = 112px 常驻。

优势

  • 适配摩擦归零——选择器永远在眼前
  • "12 matches" 的实时计数本身就是社会证明
  • 对标 Lasfit 等竞品，拉开差异

权衡

  • Chrome 偏重，Hero 要顶着 112px 开始
  • 移动端适配栏需要折叠成抽屉
  • 对品牌/生活方式内容不友好——气质偏工具
Nav B · 双层适配优先栏

Nav C · 极简 + Find My Bulb 抽屉

NAV · C

顶栏只有四项：Shop · Series · Support · Learn。右上角一枚带圆点的 Find My Bulb 按钮，点击后右侧滑出一个聚焦的适配抽屉—— Year → Make → Model 三步走完，不离开当前页面。气质最接近 DJI / Aesop。

适合谁
来看品牌故事或生活方式的潜在用户——比如从 Instagram / YouTube 评测过来的流量。
核心承诺
"我们是高端的"。首屏不谈功能，只谈气质。
信息密度
低。72px 顶栏，且叠在 Hero 上浮动透明。

优势

  • Hero 视觉呼吸感最强
  • 抽屉式适配是独立模块，可复用到 PDP
  • 首页不像零件店——拉开品牌调性

权衡

  • 没有常驻 CTA，首屏转化可能降
  • 对只想快速买的老用户有点反直觉
  • 四项主导航需要信息架构完全重写
Nav C · 深色极简 + 抽屉

03 · Hero 方向：同一句承诺，三种情绪读法

三版 Hero 对外说的都是同一件事——"精准适配 + LED 升级"。差异在于情绪切入点： 是把夜路开给你看（电影感）、把产品摆成杂志封面（编辑风）、还是把 fitment 选择器直接变成主角（工具感）。

Hero 1 · Cinematic Beam 电影感光束

HERO · 1

左文字右产品。标题 "See the road. Own the dark." 用 Space Grotesk 的粗体 + 亮橙 accent；右侧是夜路实拍（目前是占位图）+ 产品参数浮层（Series / Lumens / Kelvin）。 下方嵌入一个半透明的 fitment 小部件——既给工具感，又不抢情绪。

情绪
工程派 · DJI / Tesla · 偏男性向
资产需求
一张真实夜路照片（雨天湿路 / 山区弯道），主光源在画面右侧
和谁搭
Midnight 色调 + Nav A 或 Nav C

优势

  • 品牌记忆点最强——"黑夜 + 橙色光束"
  • fitment 浮层让转化路径不丢
  • 参数 chip 让工程用户一眼读懂

权衡

  • 需要高质量夜景实拍，不能用 stock
  • 深色首屏对浅色产品区有过渡成本
  • 对"想看人用"的 UGC 路线不亲和
Hero 1 · Cinematic

Hero 2 · Editorial Split 杂志分屏

HERO · 2 ★

左文右图的杂志排版。标题混排 Space Grotesk + Instrument Serif 斜体——"Brighter. Cleaner. Finally, yours." 右侧是产品棚拍大图 + 三张系列缩略图；左下三个数字指标（300% / 10K+ / 4.7★）做底部支撑。整体温暖、编辑、有人味。

情绪
生活方式 · Velotric / Kwikset · 性别中立
资产需求
一张棚拍产品 3/4 角旗舰图 + 3 张系列缩略（目前 CSS 占位）
和谁搭
Daylight 色调 + Nav B（最搭）

优势

  • 读完第一屏用户知道有几个系列——缩短选择路径
  • serif 斜体拉出品牌感，和竞品拉开
  • 指标行既是卖点也是信任锚

权衡

  • 棚拍资产得补齐——当前占位看上去空
  • 第一屏没有 fitment——要靠 Nav B 的顶栏补
  • 浅色首屏对夜景叙事克制
Hero 2 · Editorial

Hero 3 · Interactive Fitment-first 适配优先交互式

HERO · 3

首屏正中间就是一个巨型 Year · Make · Model 选择器，下方 "Show 12 compatible bulbs" 的橙色大按钮。 背景是 loop 视频占位（夜路 + 光束扫过弯道），底部四个数字 stat（500K+ / 10K+ / 24h / 30-day）。

情绪
工具感 · 电商 · 直给
资产需求
一段 10-15 秒的无声循环视频（弯道 + 车灯扫过）
和谁搭
Midnight 色调 + Nav C（此时顶部不再显示 Fitment，避免冗余）

优势

  • 转化漏斗最短——首屏就能开始适配
  • 数字 stat 行给公信力
  • 结构最像 Lasfit 等转化型竞品

权衡

  • 品牌叙事牺牲——首屏没故事
  • 和 Nav B 功能重叠，会显得冗余
  • 视频资产要求高，加载也要兜底
Hero 3 · Fitment-first

04 · 色调系统：三种可切换的视觉情绪

色调和 Hero / Nav 独立——同一个导航可以是深色也可以是浅色。Midnight 继承 v03 的夜景感， Daylight 是对竞品杂志风的回应，Hybrid 是"首屏深 · 产品区浅 · UGC 深"的节奏组合，避免整页单调。

01 · MIDNIGHT

深色 · Cinematic

保留 v03 的 #0D0D0D + 橙 #FF6600 骨架。夜路实拍自然贴合，品牌记忆最强。DJI / Tesla 调性。

02 · DAYLIGHT

浅色 · Editorial

暖白 #F7F5F1 + 墨黑 + 橙 accent。Velotric / Kwikset 杂志风。对棚拍产品友好，性别更中立。

03 · HYBRID

混合 · 节奏组合

首屏深 → 产品系列浅 → UGC 深 → Footer 浅。避免单一色调的视觉疲劳，节奏感最强，但 token 切换成本最高。

05 · 决策矩阵：哪些组合说得通

不是所有 Nav × Hero 都配得起。适配优先栏（Nav B）配适配优先 Hero（Hero 3）会让同一个选择器出现两次； 极简导航（Nav C）配电影感 Hero（Hero 1）缺少明确 CTA。下面这张矩阵是我们实际跑过的判断。

导航 ↓ · Hero → Hero 1 · 电影感 Hero 2 · 杂志风 Hero 3 · 适配优先
A · 按需求 Mega ✓ 可行 △ 偏重 ✗ 冗余
B · 适配优先栏 ✓ 强 ✓ 推荐 ★ ✗ 功能重复
C · 极简 + Drawer △ 需补 CTA ✓ 高端 ✓ 克制

06 · 我们的推荐

首选组合 · 可作为 v04 主线

Nav B · 适配优先栏  +  Hero 2 · 杂志风  +  Daylight 色调

理由：把 SEALIGHT 最硬的"车型适配"承诺做进 chrome（Nav B），Hero 就腾出来承载品牌叙事而不是重复问 VIN（Hero 2）。 浅色 Daylight 让棚拍产品自然发光，也和 UGC 照片的暖调一致。三件事各司其职，不互相拧。

08 · 下一步

STEP 01

对齐首选组合

本周内确认是否采纳 Nav B + Hero 2 + Daylight，或给出要调整的变量。评审会可以直接用这份文档逐轴过。

STEP 02

UGC 资产征集

Ops 启动 5-10 张 Before/After 真实安装照 + 3 条安装短视频的征集，是 Hero 2 和 Reviews 模块的硬依赖。

STEP 03

Fitment API 确认

确认当前 Year-Make-Model 接口是否返回推荐系列字段。Nav B 的"12 matches"实时计数依赖这个。

STEP 04

移动端适配稿

Nav B 的双层栏在移动端需要折叠成单行 + 适配 drawer。产出一版 375px 宽的移动端细化稿。

STEP 05

设计 Token 落代码

把 tokens.css 里的三套色调变量落到现有 design system，让切换真的只是 data-tone 属性切一下。

STEP 06

A/B 测试埋点清单

首选 Nav B + Hero 2 上线后，对照 v03 做 AB。埋点关注：Hero CTA 点击、适配栏完成率、PDP 到达率。
