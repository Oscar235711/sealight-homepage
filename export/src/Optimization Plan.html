01

背景 · 问题 · 目标

本轮优化面向北美 DIY 车主、对原厂卤素不满的升级用户、以及越野/皮卡改装圈。教育入口优先级下降，转化入口优先级上升。

v03 · PAIN POINTS

当前原型的问题

  • 页面偏长——10 个模块线性堆砌，用户往下滚 3 屏就流失
  • 没有动态感 / 活人感——缺真实车辆照、缺 Before/After、UGC 只有文字
  • 缺品牌特色——橙色是唯一视觉抓手，没有形成独特语言
  • 导航结构不清——4 个顶部链接，跳转逻辑和分类逻辑互相打架
  • Hero 转化链条单薄——Fitment 选择器是全部，没有「按驾驶场景」的分流
v04 · GOALS

本轮要达成的

  • 场景清晰划分——夜驾 / 雨雾 / 安装难度 / 通用装配 四条平行入口
  • 转化优先——Fitment 作为全局常驻功能，不只是 Hero 里的 widget
  • 品牌强化——橙色 + Midnight 底色 + 单衬线强调字的系统化表达
  • 导航重排——Shop by Need · Series · Category · Support 四主栏
  • 教育降级——不在首屏；放到 Footer 附近的 Edu 模块 + 文章页承接

02

信息架构 · 导航重排

v03 的四主栏（Shop by Need / Series / Categories / Support）方向是对的，但层级和 Mega 内容需要重写。v04 把一级分类从功能型改成 「场景 + 车型」 双轴，让 DIY 用户不用懂产品就能进。

新顶栏一级结构

SEALIGHT ├─ SHOP BY NEED → 按驾驶场景入口（Mega: Night · Weather · Install · Universal · Interior） ├─ SERIES → 五系对比（Nova · Pulsar · Prism · Spark · Ray） ├─ CATEGORIES → 按灯位入口（Headlight · Fog · Brake · Signal · Interior · Combo） ├─ SUPPORT → Fitment · Install · Troubleshoot · FAQ · Contact └─ [Fitment Finder] ← 常驻 CTA · 二级 sticky bar (见 Nav B)

Nav 方案对比（v04 demo 中 Row 2 已提供原型）

方案交互核心适合场景风险
A · Mega by Need 悬停展开四列 Mega，最右列放一张 feature 图片 + 推荐系列 北美主流电商习惯（最接近 Costco / Velotric） 移动端需要完全重写成 Drawer
B · Fitment-first Sticky Bar 顶栏下方二级条常驻 Year/Make/Model 三选，滚动也不消失 Fitment 转化驱动（最接近 Amazon 汽配） 视觉噪声高；会压缩首屏 Hero 空间
C · Minimal + Drawer 顶栏只 4 个链接 + 圆形橙色 CTA；详情全靠右侧 Drawer 品牌感强（最接近 DJI / Kwikset） 信息密度低；对不熟悉 drawer 的用户摩擦大

推荐 · Nav A (Mega) + 全局 Fitment Sticky

取 A 的结构（成熟、可扩展）+ B 的 Fitment 常驻（转化刚需），放弃 C。Nav C 的极简感在 Hero 里通过 H3 的 fitment-first 布局单独保留。

04

推荐组合 · 给设计/技术的最终主张

demo 里给了 Nav ×3、Hero ×3、Tone ×3 共 27 种组合。下面是我推荐给生产的那条路径——供设计 review 时作为讨论锚点。

NAV

Nav A — Mega by Need recommended

+ Fitment sticky from Nav B

结构最成熟、最接近北美电商习惯；Mega 的四列能一次性承载 Need / Series / Category / Feature，不牺牲任何一级入口。Fitment sticky bar 从 Nav B 里剥离出来做为全局组件，保证任何深页都能触发 Fitment。

HERO

Hero 1 — Cinematic Beam recommended

fallback · Hero 3 Fitment-first

Cinematic Beam 最能解决「没有活人感」的痛点——视觉承载高，留给 Fitment widget + 双 CTA + 下方 marquee trust 足够空间。Hero 3（Fitment-first 居中）作为 A/B 测试 variant，在「转化漏斗第一步是 Fitment」这个假设上跑数据再定。Hero 2（Editorial）更适合 /nova 产品着陆页，不是首页主路。

TONE

Hybrid tone recommended

Midnight hero → Daylight product → Midnight UGC

Midnight（深底）承接夜驾情绪 + 产品 hero；Daylight（浅底）承接 Series 对比和产品规格陈列（看清楚是关键）；再 Midnight 收尾 UGC + CTA 强化品牌记忆。这是 Velotric / Kwikset / DJI 共同的节奏。

  • 观测指标：Fitment 完成率、Scene CTR、Series 页跳转、加购率、跳出率
  • 满足上线阈值后 100% 放量 · 保留 v03 作为 fallback URL 一段时间
07

Open questions · 给团队的讨论点

  1. UGC 素材节奏——Before/After + Reviews 对真实车辆照片强依赖。ops 什么时候能集齐 5-10 张高质量安装照？是否启动用户征集 + 奖励机制？
  2. Fitment 数据来源——主站 Year/Make/Model 当前返回的是否包含 series 推荐？若否，这层推荐逻辑由前端规则 or 后端新建接口？
  3. Hero 视频/动图——Hero 1 的夜驾 beam 是视频 loop 还是静态 hero？视频对加载预算的影响（LCP）需要确认。
  4. Series 命名——Nova / Pulsar / Prism / Spark / Ray 在 v03 中已确立，v04 延用。正式对外是否还叫这 5 个？有无潜在商标问题？
  5. Education 承接——首页只留 4 张 Edu 卡。深度内容是新建 /learn hub，还是挂到博客 subdomain？与 SEO 策略如何协同？
  6. 主站切换节奏——这一版是独立 staging 域名 → 一次切换，还是先把 Hero + Nav 灰度到主站？
